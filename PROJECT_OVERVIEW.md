# ResumeForge Project Overview

ResumeForge is a Python-based web application (Flask) designed to perform surgical tailoring of resumes in DOCX format based on job descriptions. It ensures the resulting resume fits on a single page by employing dynamic XML manipulation and page-count verification via PDF conversion.

## Folder Structure

```
ResumeForge/
├── app.py                  # Main Flask web application and API controller
├── launch.bat              # Script to launch the application
├── requirements.txt        # Python dependency manifest
├── setup.bat               # First-time setup script
├── engine/                 # Engine module for XML parsing & editing
│   ├── __init__.py
│   ├── master_prompt.py    # Generates the prompt used with Claude/ChatGPT
│   └── resume_engine.py    # Core DOCX XML editing and cut planning logic
├── outputs/                # Temporarily stores generated DOCX/PDF files
├── static/                 # Directory reserved for static assets (empty)
├── templates_docx/         # Input Word templates (.docx format)
│   ├── Business_Analyst.docx
│   ├── Data_Analyst.docx
│   └── Data_Scientist.docx
└── templates_html/         # Front-end UI templates (HTML/CSS)
    └── index.html
```

## Architecture Overview

ResumeForge operates on a split client-AI-server architecture:
1. **AI Prompting (Client-Side Initiation):** The user copies a structured `Master Prompt` generated by the backend and submits it along with a target Job Description to a Large Language Model (e.g., ChatGPT or Claude).
2. **JSON Input & Parsing:** The LLM produces a structured JSON output reflecting the reframed resume content. The user pastes this JSON back into the ResumeForge web UI.
3. **XML Engine Processing:** The Python engine directly parses the XML structure of the template (`word/document.xml`) using `lxml.etree` to perform exact in-place text replacements without disrupting document layouts, fonts, or styles.
4. **Overflow & Cut Planning:** The application checks the document length by converting the DOCX to a PDF. If the resume overflows beyond one page, a prioritized "cut plan" suggests trimmings to fit the resume back onto a single page.
5. **Confirmation & Download:** The user selects which cuts to apply, and the engine executes the final updates. The user can download the completed resume in `.docx` or `.pdf` format.

## Technology Stack

- **Backend:** Python 3, Flask 3.1.0
- **Document Processing:** `python-docx` 1.1.2, `lxml` 5.3.0, custom ZIP/XML manipulation
- **PDF Conversion:** `docx2pdf` (Windows MS Word COM automation wrapper), LibreOffice CLI fallback, and PowerShell COM automation fallback
- **Frontend:** Vanilla HTML5, Vanilla CSS3 (custom dark-theme layout using CSS variables, Google Fonts "Space Mono" & "DM Sans"), and native asynchronous JavaScript APIs (`fetch`)

## Build and Setup Process

1. Run `setup.bat` to install packages: `pip install flask python-docx lxml docx2pdf`.
2. Run `launch.bat` to execute `python app.py`. This starts a local server on port `5000` and automatically opens `http://localhost:5000` in the default web browser.
